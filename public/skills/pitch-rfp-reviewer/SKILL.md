# Pitch/RFP Reviewer

Evaluate pitch responses with rigorous criteria before they go out the door. Better to find weaknesses now than lose the pitch.

## Review Framework

### 1. Compliance Check

Does the submission meet basic requirements?

| Requirement | Status | Notes |
|-------------|--------|-------|
| Addresses all RFP questions | ✓/✗ | [Missing items] |
| Within page/word limits | ✓/✗ | [Current vs. limit] |
| Correct format/structure | ✓/✗ | [Issues] |
| Required documents included | ✓/✗ | [Missing items] |
| Submitted by deadline | ✓/✗ | [Risk assessment] |

**Non-compliance = automatic disqualification** in many processes. Flag immediately.

### 2. Strategic Alignment Assessment

Does the response demonstrate genuine understanding?

**Client Understanding** (Score 1-5):
- [ ] Reflects their specific situation, not generic category
- [ ] Uses their language and terminology
- [ ] Identifies unstated needs/challenges
- [ ] Shows awareness of their competitive context
- [ ] Demonstrates knowledge of their audience

**Brief Alignment** (Score 1-5):
- [ ] Addresses core objectives directly
- [ ] Scope matches what they asked for
- [ ] Timeline aligns with their needs
- [ ] Budget within their indicated range
- [ ] Evaluation criteria addressed

**Strategic Depth** (Score 1-5):
- [ ] Clear strategic framework/approach
- [ ] Insight or perspective that adds value
- [ ] Connection between strategy and execution
- [ ] Measurable outcomes defined
- [ ] Risk awareness demonstrated

### 3. Persuasiveness Evaluation

Will this convince the decision-makers?

**Opening Impact** (Score 1-5):
- [ ] Executive summary compelling standalone
- [ ] Clear value proposition within first page
- [ ] Hook that creates interest
- [ ] Confidence without arrogance

**Evidence Quality** (Score 1-5):
- [ ] Case studies relevant to opportunity
- [ ] Results quantified where possible
- [ ] Testimonials or references included
- [ ] Team credentials match scope
- [ ] Claims supported by proof

**Competitive Positioning** (Score 1-5):
- [ ] Clear differentiation articulated
- [ ] "Why us" answered convincingly
- [ ] Unique capabilities highlighted
- [ ] Potential objections addressed

**Clarity & Readability** (Score 1-5):
- [ ] Structure easy to navigate
- [ ] Key points scannable
- [ ] Jargon-free or jargon-appropriate
- [ ] Visual design professional
- [ ] Error-free (typos, grammar)

### 4. Completeness Audit

What's missing or thin?

**Gaps to flag**:
- Questions unanswered or partially answered
- Sections underdeveloped
- Evidence missing for claims
- Team gaps for scope proposed
- Timeline unrealistic
- Budget unexplained

**Enhancement opportunities**:
- Where could we go deeper?
- What additional proof points available?
- What objections aren't addressed?
- Where is competition likely stronger?

### 6. Red Flags

Issues that could lose the pitch:

**Critical** (must fix):
- [ ] Non-compliance with requirements
- [ ] Client name wrong or inconsistent
- [ ] Major strategic misalignment
- [ ] Pricing significantly off-market
- [ ] Claims without evidence

**Significant** (should fix):
- [ ] Weak opening/executive summary
- [ ] Irrelevant case studies
- [ ] Generic rather than tailored
- [ ] Key questions underanswered
- [ ] Team doesn't match scope

**Minor** (nice to fix):
- [ ] Typos or grammatical errors
- [ ] Design inconsistencies
- [ ] Verbose sections
- [ ] Missing polish elements

## Quick Evaluation Checklist

For rapid review when time is short:

**Non-negotiables**:
- [ ] Compliant with requirements
- [ ] No errors in client name
- [ ] Executive summary compelling
- [ ] Questions all answered
- [ ] Pricing included and clear

**Quality markers**:
- [ ] Shows genuine understanding
- [ ] Strategic thinking evident
- [ ] Relevant case studies
- [ ] Team matched to scope
- [ ] Clear differentiation

**Polish**:
- [ ] Error-free
- [ ] Professionally designed
- [ ] Easy to navigate
- [ ] Appropriate length

## Scoring Rubric

**5 = Excellent**: Exceeds expectations, competitive advantage
**4 = Good**: Meets expectations well, minor improvements possible
**3 = Adequate**: Meets minimum requirements, notable gaps
**2 = Weak**: Below expectations, significant issues
**1 = Poor**: Does not meet requirements, fundamental problems

## Quality Checklist for Review

- [ ] Reviewed against RFP requirements point-by-point
- [ ] Scored against evaluation criteria
- [ ] Strengths and weaknesses identified
- [ ] Specific improvement recommendations made
- [ ] Red flags clearly flagged
- [ ] Competitive positioning assessed
- [ ] Timeline to implement fixes considered
